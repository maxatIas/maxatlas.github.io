# 404 Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a custom cyberpunk 404 page with a split-hero layout (cover image left, "Wrong Warp Pipe" arcade message right) that integrates seamlessly with the existing maxatlas Hugo theme.

**Architecture:** Hugo's `404.html` template with a page-bundle at `content/404/` so the user can drop a `cover.*` image alongside the content. New `.page-404` CSS block appended to `main.css`, reusing existing design tokens. No new tokens, no new partials.

**Tech Stack:** Hugo, Go templates, CSS custom properties. Dev server: `hugo server --buildDrafts --buildFuture` at `http://localhost:1313/`.

### Task 3: Create the `404.html` Hugo template

**Files:**
- Create: `themes/maxatlas/layouts/404.html`

- [ ] **Step 1: Create `themes/maxatlas/layouts/404.html`**

```html
{{ define "main" }}
{{ $cover := .Resources.GetMatch "cover.*" }}
<div class="page-404">
  <div class="page-404__hero{{ if not $cover }} page-404__hero--no-cover{{ end }}">
    {{ if $cover }}
    <div class="page-404__cover">
      <img src="{{ $cover.RelPermalink }}" alt="404">
      <div class="article__cover-scanlines"></div>
    </div>
    {{ end }}
    <div class="page-404__body">
      <div class="page-404__code">404</div>
      <div class="page-404__sub">WRONG WARP PIPE</div>
      <p class="page-404__msg">
        This level does not exist.<br>
        The plumber is <em>somewhere</em>, but not here.<br>
        No checkpoints. No respawn.
      </p>
      <a href="{{ "/" | relURL }}" class="page-404__cta">← Find a valid warp</a>
    </div>
  </div>
</div>
{{ end }}
```

- [ ] **Step 2: Verify the page renders**

Visit `http://localhost:1313/404.html` in the browser.

Expected:
- Nav bar at top (from `baseof.html`)
- Split hero: cover area on the left shows the grid placeholder (no cover image yet), message on the right
- "404" in large cyan neon text
- "WRONG WARP PIPE" in magenta below
- Message body in light grey
- "← Find a valid warp" button in cyan
- Clicking the CTA navigates to `/`

If the cover column appears but is empty/blank — that's correct, no `cover.*` exists yet.

- [ ] **Step 3: Verify mobile layout**

Open browser DevTools → toggle device mode at 400px wide.

Expected:
- Cover column stacks above the body (flex-direction: column)
- Cover becomes full width, fixed 180px height
- Body padding reduces

- [ ] **Step 4: Commit**

```bash

git commit -m "feat: add custom 404 page — wrong warp pipe arcade style"
```
